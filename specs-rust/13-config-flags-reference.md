# 13 — Config Flag Reference (verbatim from `config/`)

> **Status:** Conforms to `00-overview-and-conventions.md` §7.5 and
> `12-node-config-api-wallet.md` §1 (both binding). This file is the **exhaustive,
> verbatim** flag catalogue extracted from the Go tree at
> `config/{keys.go,flags.go,config.go,viper.go}`. Every value below is resolved to
> its concrete value; constants are dereferenced to their Go source. Where a default
> is genuinely network-dependent or runtime-derived, it is flagged explicitly.
>
> Source of truth: `config/keys.go` (key → flag string), `config/flags.go`
> (`addProcessFlags` + `addNodeFlags`: type/default/description), `config/config.go`
> (`GetNodeConfig` + helpers: derived/validated/network-dependent defaults),
> `config/viper.go` (env-var derivation). **206 flags total** (2 process + 204 node):
> `keys.go` declares 205 keys in its `const` block plus `HTTPWriteTimeoutKey`

## 0. Precedence, env vars, config file, deprecation

### Precedence (highest wins) — `config/viper.go`, mirrors `spf13/viper`

```
explicit CLI flag  >  env var (AVAGO_*)  >  config file  >  built-in default
```

In the Rust `Layered` resolver (`12` §1.5): `ArgMatches::value_source ==
CommandLine` → CLI; else `env.get(key)`; else `file_lookup(key)`; else
`FlagSpec::default`. Viper's `IsSet(key)` is true if the key was provided at any
non-default layer — this is what powers all the "Ignored if X is specified" /
"only one of …" rules below; reproduce it as `Layered::is_set`.

### Env-var derivation — `config/viper.go`

```go
const EnvPrefix = "avago"
DashesToUnderscores = strings.NewReplacer("-", "_")
EnvVarName = strings.ToUpper(prefix + "_" + DashesToUnderscores.Replace(key))
// viper: AutomaticEnv() + SetEnvKeyReplacer("-"→"_") + SetEnvPrefix("avago")
```

Rule: `AVAGO_` + `UPPER( key with '-' → '_' )`. Examples: `network-id` →
`AVAGO_NETWORK_ID`; `http-port` → `AVAGO_HTTP_PORT`;
`network-tls-key-log-file-unsafe` → `AVAGO_NETWORK_TLS_KEY_LOG_FILE_UNSAFE`. The
Env-var column below is generated by this rule for every row (omitted per-row only
to save space where mechanical; always recoverable from the flag name).

### Config file — `addNodeFlags`

- `--config-file <path>` — JSON/YAML/TOML config file. **Ignored if
  `--config-file-content` is specified.**
- `--config-file-content <base64>` — base64-encoded config content (takes priority
  over the path flag).
- `--config-file-content-type <json|yaml|toml>` — default `json`. Selects the
  parser for the base64 content.

Many other flags follow the same **`<x>` path / `<x>-content` base64** pair, where
the `-content` form wins via `IsSet` (genesis, upgrade, db-config, http TLS
key/cert, staking TLS key/cert, staking signer key, chain-config, subnet-config,
vm-aliases, chain-aliases).

### Path expansion — `config/flags.go::getExpandedArg`

Path-typed values run through `os.Expand`. `$AVALANCHEGO_DATA_DIR` expands to the
resolved `--data-dir` value (itself defaulting to `$HOME/.avalanchego`); any other
`$VAR` expands via the OS environment. Default paths below are shown with
`<data>` = the expanded data-dir.

### Deprecation — `config/config.go::deprecatedKeys` + `deprecateFlags`

Two keys are marked deprecated via `pflag.MarkDeprecated` (CLI prints
`Flag --<key> has been deprecated, <msg>`):

| Deprecated key | Message |
|---|---|
| `system-tracker-disk-required-available-space` | `Use system-tracker-disk-required-available-space-percentage instead` |
| `system-tracker-disk-warning-threshold-available-space` | `Use system-tracker-disk-warning-threshold-available-space-percentage instead`* |

\* Go composes the message as `Use <SystemTrackerWarningAvailableDiskSpacePercentageKey> instead`. Both deprecated keys are still
registered (type `u64`, defaults shown below) and still parsed, but the node uses
the percentage-based replacements; the byte-based ones are otherwise inert.

There are **no hidden flags** (no `MarkHidden` calls). `network-tls-key-log-file-unsafe`
is a debugging-only flag (not deprecated/hidden, but documented "Should only be
specified for debugging").

### Resolved-constant legend

`KiB=1024`, `MiB=1 048 576`, `GiB=1 073 741 824`,
`DefaultMaxMessageSize = 2·MiB = 2 097 152`.
AVAX units: `NanoAvax=1`, `MicroAvax=1 000`, `MilliAvax=1 000 000`,
`Avax=1 000 000 000`, `KiloAvax=1e12`, `MegaAvax=1e15`.
`reward.PercentDenominator = 1 000 000`.

**Network-dependent / validation (`getStakingConfig`):**
- On **Mainnet/Fuji**, the entire staking-economics block (`uptime-requirement`,
  the stake amounts/durations, reward rates, `min-delegation-fee`) is **ignored**;
  `genesis.GetStakingConfig(networkID)` is used instead. The flags apply only on
  non-standard networks, where they are validated:
  `uptime ∈ [0,1]`; `min ≤ max validator stake`; `min-delegation-fee ≤ 1 000 000`;
  `min-stake-duration > 0`; `max ≥ min stake duration`;
  `max-consumption ≤ PercentDenominator`; `max-consumption ≥ min-consumption`;
  `minting-period ≥ max-stake-duration`.
- `sybil-protection-enabled=false` is **rejected on Mainnet/Fuji**
  (`errSybilProtectionDisabledOnPublicNetwork`); also requires
  `sybil-protection-disabled-weight > 0` (`errSybilProtectionDisabledStakerWeights`).
- Staking signer: **at most one** of `staking-ephemeral-signer-enabled`,
  `staking-signer-key-file-content`, `staking-signer-key-file`,
  `staking-rpc-signer-endpoint` (`errInvalidSignerConfig`).
- Staking TLS: `*-key-file-content` and `*-cert-file-content` must be set together
  (`errStakingKeyContentUnset`/`errStakingCertContentUnset`); ephemeral cert flag
  short-circuits file/content loading.

### File-based subnet config (`--subnet-config-dir` / `--subnet-config-content`)

Per-subnet file path: `<subnet-config-dir>/<subnetID>.json` (extension `.json`).
The base64 `--subnet-config-content` form is a JSON **map** `{ "<subnetID>": {…} }`.
Each value unmarshals into `subnets.Config` (`subnets/config.go`):

```jsonc
{
  "validatorOnly": false,                  // bool; default false
  "allowedNodes": ["NodeID-..."],          // set; ONLY valid when validatorOnly=true
  "proposerNumHistoricalBlocks": 0,        // uint64; default proposervm.DefaultNumHistoricalBlocks = 0
  // exactly ONE of the following three consensus blocks (else ErrTooManyConsensusParameters):
  "snowParameters":    { "k":20, "alphaPreference":15, "alphaConfidence":15, "beta":20,
                         "concurrentRepolls":4, "optimalProcessing":10,
                         "maxOutstandingItems":256, "maxItemProcessingTime":"30s" },
  "simplexParameters": { "maxNetworkDelay":"5s", "maxRebroadcastWait":"5s" },
  "consensusParameters": { /* DEPRECATED alias of snowParameters; see below */ }
}
```

Loading & defaulting (`getSubnetConfigFromBytes` → `applySubnetConfigDefaults`):
- Start from `getPrimaryNetworkConfig(v)` (i.e. the top-level snow flags), clear the
  three consensus-param pointers, then JSON-unmarshal the file over it.
- `ValidConsensusConfiguration()`: **at most one** of `consensusParameters`,
  `snowParameters`, `simplexParameters` (`ErrTooManyConsensusParameters`).
- Consensus mode resolution (`resolveConsensusMode`):
  - `simplexParameters` set → fill its unset (zero-valued) fields from
    `--simplex-max-network-delay`/`--simplex-max-rebroadcast-wait`.
  - `snowParameters` set → fill unset fields from the top-level snow flags
    (`applySnowballParameterDefaults`; a zero field is treated as unset; the
    deprecated `alpha` field, if present, sets both `alphaPreference` and
    `alphaConfidence`).
  - `consensusParameters` set (DEPRECATED) → default it, then **migrate** it into
    `snowParameters` and null out `consensusParameters`.
  - none set → `snowParameters = getPrimaryNetworkSnowConfig(v)` (top-level flags).
- `ValidParameters()`: `allowedNodes` non-empty requires `validatorOnly=true`
  (`errAllowedNodesWhenNotValidatorOnly`); then `snow`/`simplex` `Verify()`.
- The **Primary Network** subnet config is always
  `getPrimaryNetworkConfig(v)` (`validatorOnly=false`,
  `proposerNumHistoricalBlocks=0`, snow params from the top-level flags) and is
  validated separately.

> **Task-spec note:** the prompt mentions `gossipConfig` and
> `proposerMinBlockDelay` per subnet. In the **current** Go source these are **not**
> fields of `subnets.Config`. `proposerMinBlockDelay` exists only as the
> node-global `--proposervm-min-block-delay` flag (`nodeConfig.ProposerMinBlockDelay`),
> and there is no per-subnet `gossipConfig`. The only per-subnet proposervm field is
> `proposerNumHistoricalBlocks`. Implementers should match the schema above, not the
> prompt's list.

### File-based chain config (`--chain-config-dir` / `--chain-config-content`)

Per-chain directory layout: `<chain-config-dir>/<chainAlias>/config.<ext>` and
`<chain-config-dir>/<chainAlias>/upgrade.<ext>` (read by
`storage.ReadFileWithName`, extension-agnostic). Each `<chainAlias>` subdirectory
yields `chains.ChainConfig{ Config: []byte, Upgrade: []byte }` keyed by the
directory name (`readChainConfigPath`). The base64 `--chain-config-content` form is

If `--chain-config-dir`/`--subnet-config-dir` is **explicitly set** but the dir
does not exist → `errCannotReadDirectory`; if **unset** and absent → silently

### Alias-file format (`--chain-aliases-file`, `--vm-aliases-file`)

JSON object mapping an ID (cb58 string) to a list of alias strings:
`map[ids.ID][]string`, e.g. `{ "<blockchainID>": ["MyChain","mychain"] }`
(`getAliases`). If the path is explicitly set but missing → `errFileDoesNotExist`;
if unset and missing → `nil` (no aliases).

## 25. Parity test (golden snapshot)

Per `12` §1.8 and `02`:

1. **Generator (Go side, xtask `gen-flags`):** iterate `config.BuildFlagSet()`
   (`fs.VisitAll`) and dump, sorted by name, one record per flag:
   `{ "name", "type", "default", "deprecated", "deprecation_msg" }` to
   `tests/golden/flags.json`. `default` is the pflag `DefValue` string; `type` is
   the pflag `Value.Type()` (`string`,`bool`,`int`,`uint`,`uint64`,`float64`,
   `duration`,`intSlice`,`stringSlice`,`stringToString`).
   - For runtime-derived CPU defaults and the OS-dependent `fd-limit`, emit the
     symbolic form (or run the generator with `GOMAXPROCS`/`GOOS` pinned and record
     the pinned environment in the snapshot header) so the comparison is stable.

2. **Rust side:** `build_command(&FLAG_SPECS)` enumerates `FlagSpec`s; serialize the
   same record shape and assert:
   - **set-equality of names** (catches added/removed/renamed flags);
   - **per-flag `type` equality** (mapping `FlagKind` → the Go type string above);
   - **per-flag `default` string equality** (normalize Duration via
     `parse_go_duration` round-trip; normalize the network-dependent / runtime
     defaults by the symbolic-form rule);
   - **deprecated set + message equality**.

3. **CI gate:** the test fails on any drift in either direction (Go added a flag,
