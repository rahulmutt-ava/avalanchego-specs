# 22 — Test Vectors & the Differential Oracle (Executable Backbone)

> **Status:** Binding for every crate that ships a golden/conformance test.
> Conforms to `00-overview-and-conventions.md`. This spec is the **executable
> backbone** that turns each subsystem spec's "test plan" into a runnable
> corpus. `02-testing-strategy.md` defines the *layers* (unit → property →
> golden/conformance → integration → differential) and the *mechanism*
> (`tests/vectors/<surface>/`, provenance notes, the `vectors-drift` CI gate);
> this spec defines the **concrete corpus layout, file formats, the Go
> extraction harness, the manifest, the Rust loader, and the coverage matrix**
> that realize that mechanism. Where `02 §6`/`§13.2` set a mandate, this spec
> only refines and operationalizes it — it never waives it.

The deliverable here is a single source of truth — `tests/vectors/` plus a
deterministic Go generator (`tools/vectorgen/`) and a `cargo xtask vectors`
front end — that:

1. is **lifted from or generated by the real avalanchego code** so it cannot
   drift away from the Go protocol silently (the `vectors-drift` gate, `02 §6.2`),
2. is **consumed identically by every Rust crate** via one generic loader, and
3. cleanly separates the **static-vector oracle** (fast, hermetic) from the
   **live-binary oracle** (`02 §11`), with a rule for which surfaces need both.

## 0. Go source paths this spec covers (source of truth)

The corpus is extracted from these Go testing artifacts; each is the provenance
for one or more vector categories (full mapping in §8).

| Go artifact | Vectors it grounds | Lift vs. generate |
|---|---|---|
| `codec/linearcodec` (`Marshal`/`Unmarshal`), per-VM `codec.go` registration order | `codec`, `proto-wire` typeID tables | **generate** (call `Marshal` on fixed inputs) |
| `utils/sampler/rand_test.go` (gonum `prng.MT19937`/`MT19937_64`), `utils/sampler/{uniform,weighted}_test.go` | `sampler-rng` | **lift** the `expected` tables + **generate** longer streams |
| `utils/crypto/secp256k1/rfc6979_test.go`, `secp256k1_test.go` | `crypto-secp` (RFC6979 det-sig, recover, low-S, addr) | **lift** RFC6979 `rsHex` + **generate** recover/addr |
| `utils/crypto/bls/{signature,public}_test.go` | `crypto-bls` (sign, aggregate, PoP) | **generate** (seeded keys → sign/agg) |
| `utils/cb58`, `utils/formatting/address`, `ids` | `address` (cb58/bech32/hex, NodeID) | **lift** existing fixtures + **generate** per-HRP |
| `x/merkledb` (`db_test.go`, `proof_test.go`) | `merkle-roots` (roots, proofs, range-proofs) | **generate** (apply op set → root/proof bytes) |
| `message/`, `network/` framing (`len_be ‖ proto_bytes`) | `p2p-framing` | **generate** per op-code |
| `vms/platformvm/txs` builders + `block` codec | `blocks-txs-pchain` | **generate** (seeded tx builders) |
| `vms/avm/txs` builders + `nftfx`/`propertyfx` | `blocks-txs-xchain` | **generate** |
| reexecute `blockexport` + Firewood-ethhash roots (`10`/`04`) | `evm-state-roots` | **lift** recorded ranges + **generate** roots |
| `vms/platformvm/warp` (`signer.go` `Sign`, `signature.go` `Verify`/`VerifyWeight`, `message_test.go`) | `warp` | **generate** (seeded signer + aggregate) |
| `vms/platformvm/txs/fee`, `vms/platformvm/validators/fee/fee_test.go`, `reward.Calculator` | `fee-math` | **lift** `*_test.go` golden tables + **generate** sweeps |
| `genesis/genesis.go` (`FromConfig` → `([]byte, ids.ID)`), `genesis/genesis_test.go` | `genesis-ids` | **generate** for Mainnet/Fuji/local |

> **Lift** = copy an existing Go table/fixture verbatim (its inputs already
> encode known bytes; we re-emit them as JSON). **Generate** = run real Go code
> on canonical, seeded inputs to *produce* the bytes. Either way the vector's
> provenance (§4) cites the exact Go symbol, and the `vectors-drift` job
> (§7) re-runs the generator to prove the bytes still match the pinned Go tree.

## 1. Corpus layout & taxonomy

All vectors live under the workspace-root `tests/vectors/`, committed to git
(`02 §6.1`). One directory **category** per protocol surface; the category name
is the key everything else (manifest, loader, coverage matrix) joins on.

```
tests/
├── vectors/
│   ├── manifest.json                 # top-level index + provenance + hashes (§4)
│   ├── codec/                         # linear-codec encodings + typeID tables
│   │   ├── typeids.json
│   │   ├── utxo.json
│   │   └── ...
│   ├── crypto-secp/                   # RFC6979 det-sig, recover, low-S, addr-from-pubkey
│   ├── crypto-bls/                    # sign, aggregate, proof-of-possession
│   ├── sampler-rng/                   # gonum MT19937 / MT19937-64 streams + sampler indices
│   ├── address/                       # cb58 / bech32 / hex / NodeID-from-cert
│   ├── merkle-roots/                  # merkledb roots, proofs, range-proofs (SHA-256)
│   ├── p2p-framing/                   # one record per message op-code (len_be ‖ proto)
│   ├── blocks-txs-pchain/             # P-Chain tx + block bytes/hashes
│   ├── blocks-txs-xchain/             # X-Chain tx/op + block bytes/hashes
│   ├── evm-state-roots/              # Firewood-ethhash state roots over recorded ranges
│   │   ├── range-0001.bin            # block-export payload (binary)
│   │   └── range-0001.json           # sidecar: start state, expected roots (§3.2)
│   ├── warp/                          # UnsignedMessage, BLS Sign, AggregateSignatures, Verify
│   ├── fee-math/                      # dynamic-fee, complexity, base-fee window, staking reward
│   ├── genesis-ids/                   # Mainnet/Fuji/local genesis bytes + block IDs
│   └── proto-wire/                    # protobuf wire bytes for shared messages (vm.proto, p2p.proto)
└── differential/proptest-regressions/ # committed seeds (02 §4.1, §11.5) — NOT vectors
```

Thirteen categories: `codec`, `crypto-secp`, `crypto-bls`, `sampler-rng`,
`address`, `merkle-roots`, `p2p-framing`, `blocks-txs-pchain`,
`blocks-txs-xchain`, `evm-state-roots`, `warp`, `fee-math`, `genesis-ids`,
`proto-wire` (= **14** with `proto-wire`; consensus uses `sampler-rng` +
`p2p-framing`; SAE block hashes nest under a `saevm/` subtree of
`blocks-txs-*`-style files but reuse the same record format).

### 1.1 File format — the self-describing JSON record (default)

Every category is a directory of `.json` files. Each file is a **vector set**: a
small header plus a `vectors` array of self-describing records. All binary
fields are lowercase hex (no `0x`), so the file is diffable and the loader is
generic. The canonical record schema (`tests/vectors/schema/record.schema.json`,
JSON-Schema draft 2020-12, also committed):

```json
{
  "category": "codec",
  "go_source": "codec/linearcodec + vms/components/avax/utxo.go",
  "codec_version": 0,
  "vectors": [
    {
      "name": "utxo/min",
      "desc": "minimal UTXO, secp256k1 transfer output, 1 owner",
      "input": {
        "tx_id": "0000...0000",
        "output_index": 0,
        "asset_id": "21e6...avax",
        "amount": 1000000000,
        "addrs": ["c3344128e060128ede3523a24a461c8943ab0859"]
      },
      "output_hex": "0000000000000000...",
      "output_id": "2Qouv...cb58"
    }
  ]
}
```

- `input` is a **typed JSON object** the Rust loader deserializes into the same
  domain type the Rust code under test produces — so the test reconstructs the
  value, re-encodes, and asserts `hex(encode(value)) == output_hex` (round-trip
  *and* byte-exactness in one record).
- `output_hex` is the load-bearing field for byte-exact surfaces; `output_id`
  (a cb58/hex ID) is added where Go also exposes a hash/ID (block IDs, UTXO IDs,
  merkle roots) so a Rust impl can assert the hash without re-deriving it.
- `desc` is human prose for review; `name` is a stable key used in test failure
  messages and in the manifest's per-record hash list (optional, §4).

### 1.2 File format — `.bin` + `.json` sidecar (large/binary payloads)

For categories whose inputs are large recorded blobs (`evm-state-roots`
block-export ranges, multi-MB block streams), the input lives in a sibling
`.bin` and the `.json` is a **sidecar** that references it and carries the
expected outputs:

```json
{
  "category": "evm-state-roots",
  "go_source": "tests/reexecute/c + blockexport; firewood ethhash root",
  "vectors": [
    {
      "name": "cchain/mainnet/0..1000",
      "input_bin": "range-0001.bin",
      "input_bin_sha256": "9f86d0818884...",
      "start_state_root": "56e81f17...keccak-empty",
      "expected_roots": [
        { "height": 1, "state_root": "d7f8974f...", "receipts_root": "..." },
        { "height": 1000, "state_root": "..." }
      ]
    }
  ]
}
```

Block export reuses the Go `blockexport` artifact bytes verbatim (`02 §10.5`),
so the `.bin` is exactly what the Go reexecute harness consumes.

## 2. The manifest (`tests/vectors/manifest.json`)

A single top-level file indexes every category and file, recording **provenance
and a content hash** so drift is detectable both ways: a Go change that re-emits
different bytes is caught by re-hashing during regeneration; an *uncommitted*
hand edit to a vector file is caught by `cargo xtask vectors verify`.

### 2.1 Schema

```json
{
  "schema_version": 1,
  "avalanchego_revision": "fb174e8925...",
  "generated_by": "tools/vectorgen",
  "generated_at_commit_only": true,
  "categories": {
    "codec": {
      "oracle_mode": "static+live",
      "milestone": "M0",
      "files": [
        {
          "path": "codec/utxo.json",
          "sha256": "5e884898da28...",
          "provenance": {
            "go_package": "github.com/ava-labs/avalanchego/codec/linearcodec",
            "go_symbols": ["Manager.Marshal", "vms/components/avax.UTXO"],
            "emitter": "vectorgen/emit_codec.go:emitUTXO",
            "kind": "generated"
          },
          "record_count": 12
        }
      ]
    },
    "sampler-rng": {
      "oracle_mode": "static",
      "milestone": "M0",
      "files": [ { "path": "sampler-rng/mt19937.json", "sha256": "...",
        "provenance": { "go_package": "gonum.org/v1/gonum/mathext/prng",
          "go_symbols": ["MT19937.Uint64"],
          "emitter": "vectorgen/emit_sampler.go:emitMT",
          "kind": "generated" }, "record_count": 6 } ]
    }
  }
}
```

Fields:
- `avalanchego_revision` — the **pinned** Go commit the corpus was extracted
  from. The `vectors-drift` job re-runs `vectorgen` against *this* revision; if
  the resulting bytes differ from the committed files, CI fails (§7).
- `oracle_mode` ∈ `static` | `live` | `static+live` (§5).
- `milestone` — the `16` milestone that introduces the category (§8 matrix).
- `provenance.kind` ∈ `generated` | `lifted` (§0).
- `sha256` — content hash of the file as committed (whole-file, after canonical
  JSON formatting with sorted keys + 2-space indent so hashing is stable).

### 2.2 `cargo xtask vectors verify`

A workspace `xtask` subcommand (mirrors `02 §1`'s `cargo xtask` surface) that
**does not need the Go tree** — it validates internal consistency:

```text
cargo xtask vectors verify
  1. parse manifest.json; assert schema_version is supported
  2. for every category dir on disk: assert it appears in the manifest (no orphans)
     and vice-versa (no missing files)
  3. for every file: recompute sha256 over canonical-formatted bytes;
     assert == manifest.sha256        → catches uncommitted hand edits
  4. validate every record against tests/vectors/schema/record.schema.json
  5. assert oracle_mode ∈ allowed set and every consensus-critical category
     (see §5) is "static+live"
  → exit non-zero on any mismatch; print the offending path + diff
```

`cargo xtask vectors regen` (re-extract, §3/§7) is the *only* sanctioned way to
change a vector or its hash; `verify` is a pure check run on every PR.

## 3. The Go extraction harness — `tools/vectorgen/`

A small, dependency-light Go program that **imports avalanchego packages and
emits the corpus deterministically**. It is the executable embodiment of `02
§6.2`. It lives in the avalanchego tree (or a sibling module pinned to a
revision) so it links the *real* code:
`linearcodec.NewDefault().Marshal`, `bls.SecretKeyToBytes`/`bls.Sign`,
`gonum prng.MT19937`, `genesis.FromConfig`, `platformvm` tx builders,
`fee` calculators, warp `NewSigner().Sign` / `AggregateSignatures`.

### 3.1 Structure

```
tools/vectorgen/
├── go.mod                       # pinned to the avalanchego revision in manifest
├── main.go                      # CLI: --out tests/vectors --only codec,sampler-rng ...
├── emit.go                      # Emitter interface + record/manifest writers
├── emit_codec.go                # category: codec, proto-wire
├── emit_crypto.go               # crypto-secp, crypto-bls, address (nodeid)
├── emit_sampler.go              # sampler-rng (gonum MT19937 / -64, sampler indices)
├── emit_merkle.go               # merkle-roots (x/merkledb)
├── emit_p2p.go                  # p2p-framing
├── emit_blocks.go               # blocks-txs-pchain, blocks-txs-xchain
├── emit_evm.go                  # evm-state-roots (reuses blockexport)
├── emit_warp.go                 # warp
├── emit_fee.go                  # fee-math (lifts the *_test.go golden tables)
└── emit_genesis.go              # genesis-ids
```

`main.go` walks a registry of emitters, each implementing:

```go
// emit.go
type Emitter interface {
    Category() string                 // e.g. "codec"
    OracleMode() string               // "static" | "live" | "static+live"
    Milestone() string                // "M0".. for the manifest (ref spec 16)
    Emit(w *Writer) error             // produce one or more vector files
}

// Writer canonicalizes JSON (sorted keys, 2-space indent), tracks sha256 per
// file, and accumulates manifest entries with provenance.
```

**Determinism rules (mandatory):** every emitter is **seeded** with a fixed
constant (no `time`, no `crypto/rand` — keys come from a fixed byte seed via a
deterministic KDF), iterates inputs in a **sorted, stable order**, and writes
through `Writer`'s canonical formatter. Re-running `vectorgen` on the same Go
revision MUST be byte-identical (that is exactly what `vectors-drift` asserts).
It must be **pinned to the manifest's `avalanchego_revision`** and re-run in CI
to detect upstream changes (§7).

### 3.2 Representative emitters

**Codec** — call the real `linearcodec` `Marshal` on canonical typed inputs:

```go
// emit_codec.go
func (e codecEmitter) Emit(w *Writer) error {
    c := linearcodec.NewDefault()                 // same registration order as Go VM
    m := codec.NewManager(math.MaxInt32)
    if err := m.RegisterCodec(codecVersion, c); err != nil {
        return err
    }
    utxo := &avax.UTXO{
        UTXOID: avax.UTXOID{TxID: ids.Empty, OutputIndex: 0},
        Asset:  avax.Asset{ID: avaxAssetID},
        Out: &secp256k1fx.TransferOutput{
            Amt:          1_000_000_000,
            OutputOwners: secp256k1fx.OutputOwners{Threshold: 1, Addrs: fixedAddrs},
        },
    }
    b, err := m.Marshal(codecVersion, utxo)        // <- the bytes we pin
    if err != nil {
        return err
    }
    return w.Record("codec", "codec/utxo.json", Record{
        Name:      "utxo/min",
        Desc:      "minimal UTXO, secp256k1 transfer output, 1 owner",
        Input:     utxoInputJSON(utxo),            // typed JSON the Rust loader reads
        OutputHex: hex.EncodeToString(b),
        OutputID:  utxo.InputID().String(),        // cb58 ID Rust can also assert
    })
}
```

**Sampler / RNG** — emit the raw gonum MT stream (the R1 gate, `00 §11.2`):

```go
// emit_sampler.go — produces sampler-rng/mt19937.json
func (e samplerEmitter) Emit(w *Writer) error {
    for _, seed := range []uint64{0, 1, 0xDEADBEEF} {
        mt := prng.NewMT19937()                    // gonum: the authoritative source
        mt.SeedFromKeys([]uint64{seed})            // exact seeding schedule Go uses
        stream := make([]uint64, 16)
        for i := range stream {
            stream[i] = mt.Uint64()                // bit-for-bit target for Rust
        }
        // Also drive the real weighted sampler with this RNG and record indices.
        s := sampler.NewDeterministicWeighted(seedSource(seed))
        idx := sampleIndices(s, weights, /*count*/ 8)
        if err := w.Record("sampler-rng", "sampler-rng/mt19937.json", Record{
            Name:   fmt.Sprintf("mt19937/seed-%d", seed),
            Input:  map[string]any{"seed": seed, "variant": "mt19937", "n": 16},
            Output: map[string]any{"stream": stream, "indices": idx},
        }); err != nil {
            return err
        }
    }
    // ...repeat for prng.NewMT19937_64 into sampler-rng/mt19937_64.json
    return nil
}
```

**Genesis ID** — run the node's genesis generation for each network:

```go
// emit_genesis.go — produces genesis-ids/genesis.json (02 §6.2)
func (e genesisEmitter) Emit(w *Writer) error {
    for _, netID := range []uint32{constants.MainnetID, constants.FujiID, constants.LocalID} {
        cfg := genesis.GetConfig(netID)                 // embedded canonical config
        bytes, blockID, err := genesis.FromConfig(cfg)  // <- ([]byte, ids.ID, error)
        if err != nil {
            return err
        }
        sum := sha256.Sum256(bytes)
        if err := w.Record("genesis-ids", "genesis-ids/genesis.json", Record{
            Name:  "genesis/" + constants.NetworkName(netID),
            Input: map[string]any{"network_id": netID},
            Output: map[string]any{
                "genesis_sha256": hex.EncodeToString(sum[:]), // bytes are huge; pin the hash
                "block_id":       blockID.String(),           // the parity contract
            },
        }); err != nil {
            return err
        }
    }
    return nil
}
```

Warp, fee, and BLS emitters follow the same shape: call the real symbol
(`warp.NewSigner(sk, networkID, chainID).Sign(msg)`, then
`warp.AggregateSignatures`; the `fee` `Calculate`/complexity functions, lifting
the `fee_test.go` constant tables; `bls.Sign`/`AggregateSignatures`/PoP) on
seeded inputs and record the bytes.

## 4. Provenance & drift detection

Drift can occur on two axes; the manifest catches both:

1. **Upstream Go change** (the protocol moved, or a Go refactor changed bytes).
   Caught by `vectors-drift` (§7): re-run `vectorgen` at the pinned revision,
   recompute file hashes, diff against the committed corpus. A diff is either a
   **bug** (we mis-pinned) or a **deliberate protocol change** that must bump
   `avalanchego_revision` + regenerate + get reviewed as a compatibility event
   (`02 §13.8`).
2. **Local tampering** (someone hand-edited a vector to make a Rust test pass).
   Caught by `cargo xtask vectors verify` (§2.2): recompute `sha256` and compare
   to the manifest. The manifest hash is the tamper-evident seal; you cannot
   edit a vector without also regenerating (which re-derives from Go, not from
   the desired Rust output).

Every file entry carries `provenance.go_symbols` + `provenance.emitter`, so a
reviewer can trace any vector back to the exact Go call that produced it.

## 5. Static vectors vs. the live oracle

Two oracle modes, per `02 §6` and `02 §11`:

- **(a) Static committed vectors** (this spec's corpus). Fast, hermetic,
  deterministic — they run in unit/golden tests on every PR with no Go binary
  present. They pin **bytes and IDs** for pure functions (codec, hashing,
  signatures, sampler streams, genesis IDs, fee math, frames, roots).
- **(b) The live Go binary as differential oracle** (`02 §11`'s two-binary +
  recorded-oracle harness). Required for **end-to-end, stateful, multi-component
  behavior** that a static byte cannot capture: consensus finalization, mempool
  ordering, API responses, sync-to-tip, plugin interop, mixed-network no-fork.

**The rule (binding):** a surface that is *consensus-critical* — i.e. a
byte/hash that the network agrees on — gets **BOTH**. The static vector guards
the byte-exactness cheaply on every PR; the live/differential oracle guards that
the surface is *produced and accepted in context*. Pure-function-only surfaces
may be `static`; pure-orchestration surfaces (no committed bytes of their own)
may be `live`.

| Category | Oracle mode | Why |
|---|---|---|
| `codec`, `proto-wire` | `static+live` | bytes agreed on the wire; also exercised in differential tx/API flows |
| `crypto-secp`, `crypto-bls` | `static` | deterministic pure functions; verified in-context via tx/warp differential |
| `sampler-rng` | `static+live` | R1 gate (static stream) + `differential::validatorstate_parity` confirms windower on live data |
| `address` | `static` | pure derivation |
| `merkle-roots` | `static+live` | roots agreed on the `merkle/sync` wire; live in `differential::*_state_root` |
| `p2p-framing` | `static+live` | frame bytes (static) + `differential::interop_handshake` (a Go node must *accept* them) |
| `blocks-txs-pchain`/`-xchain` | `static+live` | block hashes agreed (static) + `differential::{pchain_sync_to_tip,xchain_issue_tx}` |
| `evm-state-roots` | `static+live` | recorded-oracle reexecute (`02 §10.5`) *is* the live diff over static `.bin` |
| `warp` | `static+live` | signature/message bytes (static) + verify-in-context (live) |
| `fee-math` | `static` | pure formulas; consumed inside block-building differential tests |
| `genesis-ids` | `static+live` | block ID (static) + `differential::api_parity` boot check |

`cargo xtask vectors verify` enforces that every category in the
consensus-critical set above is `static+live` (§2.2 step 5).

## 6. The Rust consumption side

### 6.1 Generic loader

One helper in a `testutil`-feature module (`ava-testvectors`, a tiny dev-only
crate, or `ava-codec::codectest` re-export) deserializes any category into typed
records. `T` is the per-category typed `input`/`output` struct the consuming
crate defines.

```rust

use serde::de::DeserializeOwned;

/// One record as stored on disk; `input`/`output` are category-typed.
#[derive(serde::Deserialize)]
pub struct Vector<I, O> {
    pub name: String,
    #[serde(default)]
    pub desc: String,
    pub input: I,
    #[serde(default)]
    pub output_hex: Option<String>,
    #[serde(default)]
    pub output_id: Option<String>,
    #[serde(flatten)]
    pub output: O,                         // extra typed outputs (e.g. {stream, indices})
}

#[derive(serde::Deserialize)]
struct VectorFile<I, O> {
    category: String,
    #[serde(default)]
    go_source: String,
    vectors: Vec<Vector<I, O>>,
}

/// Resolve `<workspace>/tests/vectors/<category>/` and load all `.json` files,
/// concatenating their records. Panics with a clear message on malformed input
/// (this is test-only code).
pub fn load_vectors<I, O>(category: &str) -> Vec<Vector<I, O>>
where
    I: DeserializeOwned,
    O: DeserializeOwned,
{
    let dir = vectors_root().join(category);
    let mut out = Vec::new();
    for entry in std::fs::read_dir(&dir).expect("category dir exists") {
        let path = entry.unwrap().path();
        if path.extension().and_then(|s| s.to_str()) != Some("json") {
            continue;
        }
        let raw = std::fs::read_to_string(&path).expect("read vector file");
        let file: VectorFile<I, O> =
            serde_json::from_str(&raw).unwrap_or_else(|e| panic!("{}: {e}", path.display()));
        assert_eq!(file.category, category, "category mismatch in {}", path.display());
        out.extend(file.vectors);
    }
    assert!(!out.is_empty(), "no vectors found for category {category}");
    out
}
```

```rust
// crates/ava-codec/tests/golden_codec.rs
use ava_testvectors::load_vectors;

#[derive(serde::Deserialize)]
struct UtxoInput { tx_id: String, output_index: u32, asset_id: String, amount: u64, addrs: Vec<String> }

#[test]
fn golden_codec_utxo() {
    // `()` output marker: this category pins bytes via `output_hex`.
    for v in load_vectors::<UtxoInput, ()>("codec") {
        let utxo = build_utxo(&v.input);                 // reconstruct the typed value
        let got = ava_codec::encode(&utxo, ava_codec::CODEC_VERSION).unwrap();
        let want = v.output_hex.as_deref().expect("codec vectors pin output_hex");
        assert_eq!(hex::encode(&got), *want, "byte mismatch: {}", v.name);

        // Round-trip closes the loop: decode(want) == value.
        let back: Utxo = ava_codec::decode(&hex::decode(want).unwrap()).unwrap();
        assert_eq!(back, utxo, "round-trip mismatch: {}", v.name);
    }
}
```

### 6.3 Integration with proptest regression corpora

Static vectors and proptest are complementary (`02 §4.3`): goldens pin the
*bytes* of curated inputs; proptest covers *structure* over generated inputs.
The two are wired together so each strengthens the other:

- **Seed the regression corpus from vectors.** A property's
  `proptest-regressions/<path>.txt` (committed, `02 §4.1`) is seeded with the
  exact inputs that appear in the matching vector file, so every golden input is
  *also* replayed under the property's invariant on every run.
- **Promote proptest failures to vectors.** When a differential/property failure
  finds a divergent input (`02 §11.5` prints `DIFFERENTIAL_SEED=<n>`), the
  minimized input is added as a named vector via `vectorgen` (extracting the
  Go-correct output) — turning a one-off failure into a permanent regression
  guard that lives in *both* corpora.

Cross-ref `02 §4` (proptest mandate), `02 §6` (golden mechanism), `02 §11.5`
(seed reproducibility).

## 7. CI integration — the vector-drift gate

Two jobs, mirroring `02 §6.2` step 4 and the `16 §4` cross-cutting workstream:

**Job `vectors-verify` (per-PR, no Go needed):**

```text
cargo xtask vectors verify     # §2.2: hashes, schema, orphans, oracle-mode rule
cargo nextest run -E 'test(golden_)'   # all golden_* tests load + assert vectors
```

**Job `vectors-drift` (per-PR, needs the pinned Go tree):**

```text
# Pin the Go module to manifest.avalanchego_revision, then regenerate in a tmp dir.
git -C $AVALANCHEGO checkout $(jq -r .avalanchego_revision tests/vectors/manifest.json)
( cd $AVALANCHEGO/tools/vectorgen && go run . --out $TMP/vectors )
cargo xtask vectors diff --against $TMP/vectors    # byte+hash diff vs committed corpus
#   → fail if anything differs: either a mis-pin (bug) or an un-regenerated change.
```

**How a deliberate protocol change updates the manifest:** the change is a
reviewed compatibility event (`02 §13.8`). The author:

1. bumps `avalanchego_revision` in `manifest.json` to the Go commit that
   introduced the change,
2. runs `cargo xtask vectors regen` (invokes `vectorgen` at the new revision,
   rewrites the affected files + their `sha256` in the manifest),
3. commits the regenerated vectors **and** the corresponding Rust impl change in
   the same PR, with the provenance note updated; reviewers diff the vector
   bytes as the explicit record of what protocol behavior moved.

This is the executable form of `02 §13.2`/`§6.2`: the corpus tracks the Go
source of truth, and a vector diff is never a silent event.

## 9. Cross-spec contract (others MUST honor)

1. **Every category named in §8 ships under `tests/vectors/<category>/`** in the
   self-describing format (§1.1/§1.2) with a manifest entry (§2) citing Go
   provenance — realizing `02 §13.2`.
2. **Every vector is produced by `tools/vectorgen/`** (generated or lifted, §0);
   no vector is hand-authored. The emitter is pinned to
   `manifest.avalanchego_revision` and is re-run by `vectors-drift` (§7).
3. **Consensus-critical categories (§5 table) are `static+live`** — a static
   byte vector *and* a `differential::*` test. `cargo xtask vectors verify`
   enforces this.
4. **Rust crates consume vectors only through `load_vectors::<I,O>(category)`**
   (§6.1) and assert byte/value equality; golden tests are named `golden_*` so
   the `vectors-verify` CI selector finds them.
5. **A vector change is a reviewed compatibility event** (§7): bump
   `avalanchego_revision`, `cargo xtask vectors regen`, commit vectors + impl +
   provenance together (`02 §13.8`).
6. **The proptest regression corpus is seeded from the matching vectors and
   grows from differential failures** (§6.3); deleting either a vector or a
   regression seed is a test regression (`02 §4.1`, §13.1).
